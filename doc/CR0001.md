# CR0001: Avoid Console.WriteLine

## Description

This rule discourages the direct use of `System.Console.WriteLine(string)` in favor of logging abstractions. Console output makes automated testing harder and is not suitable for production applications.

## Category

Usage

## Severity

Warning

## Enabled by Default

Yes

## Code Examples

### Incorrect

```csharp
using System;

public class MyClass
{
    public void LogMessage(string message)
    {
        Console.WriteLine(message); // CR0001 violation
    }
}
```

### Correct

```csharp
using Microsoft.Extensions.Logging;

public class MyClass
{
    private readonly ILogger<MyClass> _logger;

    public MyClass(ILogger<MyClass> logger)
    {
        _logger = logger;
    }

    public void LogMessage(string message)
    {
        _logger.LogInformation(message);
    }
}
```

## Why This Rule Matters

- Console output is not easily testable in unit tests.
- In production environments, logging should be configurable and directed to appropriate sinks (files, databases, etc.).
- Using logging abstractions allows for better control over log levels, formatting, and output destinations.

## Suppression

If you must use `Console.WriteLine` in specific cases (e.g., command-line tools), you can suppress this warning:

```csharp
#pragma warning disable CR0001
Console.WriteLine("This is allowed here");
#pragma warning restore CR0001
```

Or configure the rule to be disabled for specific files or projects.</content>
